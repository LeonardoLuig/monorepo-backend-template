# Purpose

This category provides reusable architectural knowledge for software engineering.

Its purpose is to establish principles, patterns, decision models, and architectural practices that support the design of maintainable, scalable, and evolvable software systems.

The knowledge contained in this category is technology-agnostic and reusable across projects.

---

# Scope

This category includes topics related to:

- Software Architecture
- System Design
- Architectural Patterns
- Architectural Principles
- Dependency Management
- Modularization
- Architectural Decision-Making

This category does not contain technology-specific implementations.

---

The following topics belong to other categories:

- Programming languages

- Infrastructure
- Business rules
- Project-specific conventions

---

# Learning Objectives

After completing this category, an engineer should be able to:

- Design maintainable software architectures.
- Evaluate architectural trade-offs.
- Define clear system boundaries.
- Organize software into cohesive modules.
- Manage dependencies appropriately.
- Apply architecture patterns based on context.
- Reason about long-term maintainability.
